### 对Vue的认识

问：Vue.是什么

> 答：官网的描述：Vue(/vjuː/)是一个渐进式JavaScript框架 特点是易用易学、灵活、高效 Vue被设计为可以自底向上逐层应用 Vue的核心库只关注视图层 不仅易于上手 还与第三方库或既有项目整合方便
> 拓展： Vue生态系统成熟 拥有Vue-router、Vuex等核心插件，VueCli、Devtools等开发工具

问：什么是渐进式框架 (了解就好)

> 答：渐进式框架 强调构建应用程序的方法是逐步增强（progressive enhancement），允许开发者根据项目的需求和复杂度选择性地使用框架提供的不同功能和特性

问：为什么Vue易学易用、灵活、高效

> 答：Vue 的数据驱动双向绑定 数据驱动视图更新 让初学者更多的关注点在视图上 减少了很多原生的dom操作 学习曲线平缓 (如果懂源码可以说说使用选项式定义数据、方法，使用指令v-model、v-bind等指令简化事件绑定 使用模版语法等)
> 在渐进式框架中 路由、状态、组件通常被设计为单独使用 根据需要单独引入 所以可以根据项目不同的需求和复杂度 选择框架满足需求的部分进行引入 或者框架不满足需求 引入其他框架 并且Vue为开发者自动优化了性能 减少了开发者的心智负担

### MVVM

问：MVVM表示的是什么

> 答：MVVM 意思是 Model-View-ViewModel

1.  Model 模型层 负责处理业务逻辑和服务端进行交互
2.  View 视图层 负责将模型和数据转化为页面展示出来 就是看到的Html
3.  ViewModel 视图模型层 用来连接视图层和模型层 是他们之间通信的桥梁

### 组件化

> 问：组件是什么 什么是组件化开发

Vue将用户界面抽象为一个个组件 每个组件都包含了自己的模版、逻辑和样式 组件可以在系统内部复用 还可以嵌套在其他组件内使用
这种组件化的开发方式使得代码更加模块化、可复用 易于维护

### 指令

问：指令是啥 常用的指令有哪些

> 答：在Vue里 指令（Directives）是指带有 v- 前缀 用于操作DOM元素的特殊属性
> 常用的指令有

1.  v-if 条件渲染指令
2.  v-for 循环/列表渲染指令
3.  v-on 事件绑定指令 简写就是@
4.  v-bind 属性绑定指令 简写就是:
5.  v-model 双向数据绑定指令

指令生命周期（进阶）

1. bind：指令第一次绑定到元素时调用，可以进行一次性的初始化设置。
2. inserted：被绑定元素插入父节点时调用，可以在元素插入到DOM中后执行相应的操作。
3. update：所在组件的VNode更新时调用，可以在元素更新时执行相应的操作。

钩子函数的参数

1.  el：指令所绑定的元素，可以直接操作DOM。
2.  binding：一个对象，包含指令的相关信息，如指令名、绑定值、修饰符等。
3.  vnode：Vue编译生成的虚拟节点。
4.  oldVnode：上一个虚拟节点。

### 和React的区别 （了解）

相同

1.  组件化思想
2.  虚拟Dom
3.  数据驱动视图更新
4.  都有自己的一套工具（测试工具、构建工具...）、插件 （路由、状态管理...）
5.  都有各自的服务端渲染框架（Vue是Nuxt.js React是Next.js）、native方案

不同

1.  数据流向 Vue是数据双向流 React是数据单向流
2.  数据是否可变 Vue推崇数据可变 React推崇数据不可变
3.  写法 Vue推崇使用template模版和单文件组件 React推崇使用JSX、高阶组件，将所有内容都放在了JavaScript中
4.  API Vue2使用的是Options API React使用hooks
5.  门槛 Vue注重降低开发门槛 React注重编程最佳实践 门槛较高
6.  diff算法不同 React主要使用diff队列保存需要更新哪些DOM，得到patch树，再统一操作批量更新DOM。Vue使用双向指针，边对比，边更新DOM

### 拓展 SPA

问：什么是SPA项目 与传统多页面项目有什么不同

> 答：SPA（Single Page Application）单页面应用程序 项目只有一个html页面 项目内的页面切换都是通过路由组件切换完成的 而不是每次切换页面都重新加载页面
>
> 好处：只需要加载一次就能加载完网站的资源 用户后续访问其他页面都是加载好了的 切换页面也只是更改了路由组件的页面 提升了用户切换页面的体验
>
> 缺点：第一次加载就要加载整个项目 首屏加载时间会更长一些 并且不同路由的html是根据虚拟dom动态生成的 不利于SEO

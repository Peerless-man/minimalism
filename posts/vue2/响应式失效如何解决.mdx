# 响应式失效出现的原因

大家在做项目时 会经常遇到在表格当前行内编辑的情况 很多时候遇到编辑当前行 但是视图没有更新的情况

# 造成响应式数据失效可能的原因 以及解决方法

1.更新当前行时 使用的下标更新 vue 响应式对于直接更改数组长度变化、下标变化不做任何处理 所以不知道数组变化了 也就无法通知页面更新

解决方法：vue 对数组的响应式进行处理时 对数组的 push、pop、shift、unShift、splice、sort、reverse 方法进行了拦截 所以在修改当前行时 可以使用 splice 替换当前行 尽量使用这些被 vue 拦截过的方法操作数组数据
[数组更新检测](https://v2.cn.vuejs.org/v2/guide/list.html#%E6%95%B0%E7%BB%84%E6%9B%B4%E6%96%B0%E6%A3%80%E6%B5%8B)

2.更新当前行时 增加了未知的属性 未知的属性没有通过 vue 的响应式处理 所以修改数据时 vue 无法更改数据 以及通知页面更新

答：由于新增了未知属性 所以 vue 无法重新再对这个对象进行响应式处理 这个属性的变化也就无法记录 可以使用$set 去将新增的属性添加入当前对象里 这个属性就会是响应式的

```js
this.$set(array, 'someAttribute', true)
```
